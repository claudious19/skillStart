# 00 Product Spec

## Scope Control

Strict

## Goal

Define the MVP product specification for SkillStart as a focused Angular and Firebase platform that connects apprenticeship graduates with companies based on visible skills, projects and potential instead of years of experience.

## Requirements

- Describe SkillStart's core promise and product positioning.
- Use the two major product roles:
  - `candidate`
  - `company`
- Mention the technical roles:
  - `candidate`
  - `company`
  - `admin`
- Describe the MVP user journeys for candidates and companies.
- State that public users can access only the landingpage.
- Define the MVP scope around:
  - landingpage
  - registration and login
  - candidate profile
  - company profile
  - job posts
  - applications
  - admin review workflow
- State that the stack is:
  - Angular
  - TypeScript
  - Firebase Authentication
  - Cloud Firestore
  - Angular Routing
  - Angular Reactive Forms
  - mobile-first UI
  - PWA-ready architecture
- State that GitHub Pages is the temporary testing deployment target.
- Emphasize that SkillStart focuses on growth, learning potential, responsibility and proof of ability.

## Non-goals

- Do not define database rules in deep technical detail.
- Do not define full visual design.
- Do not define payment or subscription implementation.
- Do not introduce matching algorithms, chat, AI scoring or analytics.
- Do not add non-MVP features from outside AGENTS.md.

## Implementation Prompt

Create a concise product specification document for SkillStart.

Follow AGENTS.md exactly.

Describe:

- the product vision
- the problem being solved
- the core promise
- the target users
- the two major product roles `candidate` and `company`
- the technical role `admin`
- the MVP scope
- the platform principles around potential, visible proof and growth-oriented jobs

Keep the document implementation-oriented so later prompts can reference it.

Do not write code.
Do not invent features outside the defined MVP.
